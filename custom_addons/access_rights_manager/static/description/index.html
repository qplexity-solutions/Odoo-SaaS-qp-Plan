Access Rights Manager

This tool allows users to customise field access rights (visible, required or read-only) and model access rights (create, edit, delete, view, actions, and duplicate) in addition to hiding menus, buttons, and tabs.In addition to making any user read-only, users can disable developer mode, hide chatter, and hide cd .. specific buttons, tabs, or menu/submenu items.



Features
  • Manage access rights for Menu Submenus
  • Manage access rights for Menu Buttons and Tabs
  • Manage access rights for models Hide Reports, Actions, Create, Update, Delete
  • Manage access rights for model fields and make any field Invisible,Required,Readonly and Hide external link
  • Disable Developer (Debug) mode
  • Manage Chatter Visibility
  • Readonly User
  • Support Community and Enterprise Editions
  • Support Multi Company
Setup

For the Access Rights Manager icon to be activated and displayed on a user form, this option must be checked.


You can now see the Access Rights icon after the user activates Access Rights Manager.

Create a new access rule.

Creating an access rule with a rule name is simple, and you can disable developer (debug mode), assign a user read-only (view only), and hide chatter from a specific user.

Assign the users to the rule you created.

Users can be assigned to your rule or the other way around.
